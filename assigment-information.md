# 📌 Guia do Trabalho Prático: Estruturas de Dados II (SCC0606)

## 📋 Informações Gerais
- **Integrantes:** 4 a 6 membros.
- **Objetivo Central:** Desenvolver um sistema adaptativo capaz de escolher automaticamente o melhor algoritmo de ordenação com base nas características da entrada.

---

## 💻 Informações sobre o Código
- [ ] **Estrutura Base:** Linguagem C, código modular, boas práticas e comentários.
- [ ] **Algoritmos Implementados:** Bubble Sort Aprimorado, Selection Sort, Merge Sort, Heap Sort, Radix Sort LSD.
- [ ] **Análise de Entrada:** Coletar tamanho, percentual de desordem, repetições, amplitude, densidade de duplicatas, padrões, grau de ordenação e distribuição.
- [ ] **Métricas Coletadas:** Tempo de execução, comparações, trocas, chamadas/profundidade de recursão, uso de memória e estabilidade.
- [ ] **CLI (Linha de Comando):** Aceitar parâmetros (ex: `--algoritmo`, `--input`).
- [ ] **Scripts de Automação:** Arquivos `.sh` ou `.bat` para compilar, executar os experimentos e gerar resultados de forma automática.
- [ ] **Testes e Entradas:**
  - [ ] Aleatórias, quase ordenadas e reversamente ordenadas.
  - [ ] Com muitos elementos repetidos.
  - [ ] Entradas de tamanho pequeno e grande.
  - [ ] **Adversariais:** Entradas feitas para degradar o desempenho ou burlar a heurística.
- [ ] **Comparação:** Testar o Sistema Adaptativo vs. Algoritmo Fixo (ex: sempre rodar Merge Sort).
